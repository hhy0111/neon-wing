# Neon Wing Run Upgrade and Ship Plan

이 문서는 한 판 안에서만 적용되는 전투 업그레이드, 기체별 미사일 차별화, 기체 상점/선택 화면 설계를 정리한다.

## 판단: 어떤 업그레이드 방식이 더 재미있는가

| 방식 | 장점 | 단점 | 판단 |
|---|---|---|---|
| 특정 적 처치 시 랜덤 아이템 드롭 | 순간 보상이 강하고 사냥감이 눈에 띈다 | 운이 나쁘면 오래 해도 성장이 막힌 느낌이 난다 | 단독 사용 비추천 |
| 점수/처치 수 도달 시 자동 업그레이드 | 공정하고 목표가 명확하다 | 예측 가능해서 전투 중 긴장감이 약하다 | 기본 성장축으로 적합 |
| 점수/처치 기반 + 특수 드롭 보너스 | 공정함과 랜덤 보상의 쾌감을 같이 준다 | UI/밸런스 설계가 조금 더 필요하다 | 출시 추천안 |

최종 결정은 `점수/처치 기반 레벨업 + 특수 적 드롭 보너스`다.
플레이어가 사냥을 많이 하면 반드시 강해지고, 엘리트/보급 드론/보스 처치 시에는 업그레이드 칩이 떨어져 순간 보상도 생긴다.
일반 적 랜덤 드롭만으로 성장시키면 운이 나쁜 판이 답답해지므로, 랜덤 드롭은 주 성장 수단이 아니라 가속/희귀 선택권으로 둔다.

## 핵심 구조

| 항목 | 설계 |
|---|---|
| 이름 | 전장 개조 / Field Upgrade |
| 적용 범위 | 현재 출격 1판에만 적용 |
| 종료 시 처리 | 사망/클리어 후 모두 초기화 |
| 영구 성장과 관계 | 격납고 업그레이드는 기본 능력치, 전장 개조는 판 안에서 쌓이는 임시 빌드 |
| 업그레이드 획득 | 사냥 점수 게이지, 엘리트/보급 드론/보스 드롭 |
| 선택 방식 | 3개 후보 중 1개 선택 |
| 한 판 목표 횟수 | 5분 기준 6~10회 |

## 출격 중 업그레이드 흐름

1. 플레이어가 선택한 기체와 기본 미사일로 출격한다.
2. 적을 처치하면 `전장 에너지`가 오른다.
3. 전장 에너지가 기준치에 도달하면 화면 하단에 업그레이드 선택 패널이 열린다.
4. 시간은 20~35% 정도 느려지고, 플레이어는 3개 후보 중 하나를 고른다.
5. 선택한 효과는 그 판에만 누적 적용된다.
6. 엘리트, 보급 드론, 보스를 처치하면 `개조 칩`이 떨어지고, 획득 시 즉시 업그레이드 선택 또는 희귀 후보 확률 증가가 발생한다.
7. 사망 또는 클리어 시 임시 업그레이드는 사라지고, 코인/보석/해금 진행도만 영구 보상으로 저장된다.

## 전장 에너지 기준

점수 기준만 쓰면 고득점 유저가 너무 빨리 강해질 수 있으므로, 점수와 처치 수를 합친 내부값을 쓴다.

```text
fieldEnergyGain = enemy.baseEnergy + scoreValue * 0.2
requiredEnergy = 100 + upgradeLevel * 65 + upgradeLevel^2 * 12
```

5분 판 기준 권장 업그레이드 타이밍:

| 업그레이드 | 예상 시간 | 목적 |
|---:|---:|---|
| 1회 | 0:25~0:40 | 초반 손맛 강화 |
| 2회 | 0:55~1:15 | 미사일 방향성 결정 |
| 3회 | 1:30~1:55 | 생존/화력 선택 |
| 4회 | 2:10~2:40 | 엘리트 대응 |
| 5회 | 2:55~3:25 | 빌드 완성 시작 |
| 6회 | 3:35~4:05 | 보스 준비 |
| 7~10회 | 숙련 플레이 보너스 | 고득점 보상 |

## 드롭 설계

| 드롭 | 출처 | 효과 | 확률/조건 |
|---|---|---|---|
| Energy Cell | 일반 적 | 전장 에너지 소량 증가 | 낮은 확률 |
| Upgrade Chip | 보급 드론, 엘리트 | 즉시 3택 업그레이드 | 보급 드론 100%, 엘리트 60~100% |
| Rare Chip | 보스 페이즈, 특수 엘리트 | 희귀 업그레이드 후보 보장 | 전장당 1~2회 |
| Repair Kit | 위험 구간 후 보너스 | HP 회복 | 낮은 확률, 광고와 별개 |
| Missile Cache | 미사일형 적/보급 드론 | 미사일 계열 후보 확률 증가 | 조건부 |

일반 적은 `Energy Cell` 위주로 둔다.
즉시 업그레이드 선택은 보급 드론과 엘리트에 집중해야 플레이어가 목표물을 인식하고 사냥하는 재미가 생긴다.

## 선택 후보 규칙

1. 항상 공격 후보 1개 이상을 보여준다.
2. HP가 35% 이하이면 방어/회복 후보 확률을 높인다.
3. 선택한 기체의 전용 미사일 업그레이드 후보를 최소 40% 확률로 포함한다.
4. 같은 업그레이드는 3~5단계까지만 중첩한다.
5. 희귀 후보는 초반 1분에는 제한하고, 2분 이후부터 확률을 높인다.
6. 보스 진입 직전에는 미사일/노바/방어 후보를 우선 노출한다.

## 상점/선택 화면 구조

### 1. 격납고 메인

목적: 현재 선택 기체, 기본 업그레이드, 출격 버튼을 빠르게 보여준다.

| 영역 | 내용 |
|---|---|
| 상단 | 보유 코인/보석, 현재 선택 기체명 |
| 중앙 | 선택 기체 큰 프리뷰 |
| 하단 탭 | 기체 선택, 업그레이드, 상점 |
| 주요 버튼 | 출격, 기체 변경, 강화 |

### 2. 기체 선택 화면

목적: 보유 기체와 잠긴 기체를 한눈에 보여준다.

| 영역 | 내용 |
|---|---|
| 좌측/상단 | 기체 카드 목록 |
| 중앙 | 큰 기체 프리뷰 |
| 우측/하단 | 기본 미사일, 패시브, 전용 업그레이드 설명 |
| 버튼 상태 | 선택됨, 선택, 구매, 조건 보기 |

잠긴 기체도 보여줘야 한다.
잠긴 기체를 누르면 구매 조건 화면이 열리고, “어떻게 얻는지”가 명확히 보여야 한다.

### 3. 기체 구매 조건 화면

목적: 기체를 살 수 있는 조건과 부족한 재화를 보여준다.

| 영역 | 내용 |
|---|---|
| 기체 프리뷰 | 잠긴 기체 이미지와 이름 |
| 조건 목록 | 전장 클리어, 처치 수, 보스 처치, 코인/보석 |
| 진행도 | 예: Shield Guard 42/80 |
| 재화 부족 표시 | 예: 코인 900/1,200 |
| 버튼 | 구매, 보석으로 해금, 돌아가기 |

구매 조건은 `조건 달성 + 재화 지불`을 기본으로 한다.
초반 기체는 조건을 가볍게, 후반 기체는 전장 진행과 처치 목표를 섞는다.

### 4. 상점 화면

목적: 실제 결제/IAP와 게임 내 재화 구매를 분리해서 보여준다.

| 섹션 | 상품 |
|---|---|
| 기체 | Astra, 스킨 후보 |
| 패키지 | 스타터 패키지, 월간 보급 패스 |
| 재화 | 보석 팩 |
| 편의 | 광고 제거 |

상점에서 직접 전투력을 과하게 팔지 않는다.
기체는 미사일 손맛과 플레이 스타일을 바꾸는 상품으로 둔다.

## 구현 우선순위

1. 웹 하네스에서 `전장 에너지` 게이지와 3택 업그레이드 선택을 먼저 구현한다.
2. 기존 미사일에 피해/재장전/추가 발사 업그레이드만 붙여 재미를 검증한다.
3. 기체 선택 화면을 추가하고 `Neon Wing`, `Raptor`, `Astra` 3종만 먼저 넣는다.
4. 기체별 기본 미사일을 다르게 만든다.
5. 구매 조건 화면과 잠금/구매 상태를 연결한다.
6. 이후 Android에 같은 데이터 구조를 이식한다.
